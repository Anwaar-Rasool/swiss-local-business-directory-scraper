# 🇨🇭 Swiss Local Business Directory Scraper

A multi-threaded Python scraper for Switzerland's largest business directory. It operates in two stages: first collecting all listing URLs by category and city, then extracting full business profiles from each listing. Includes a utility for fixing character encoding issues in saved CSVs.

---

## 📁 Project Structure

```
local-ch-scraper/
│
├── cookies.py                # Extract & serialize browser cookies
├── playwright_assistant.py   # Base Playwright browser controller (PlaywrightAssistantClass)
├── links.py                  # Stage 1: Collect listing URLs by category & city
├── local.py                  # Stage 2: Scrape full business profiles from collected URLs
├── encoding.py               # Utility: Fix mojibake/encoding issues in CSVs
│
├── cookies.pkl               # Saved browser session (auto-generated by cookies.py)
│
├── links/                    # Stage 1 output (auto-generated)
│   ├── plasterer.csv
│   ├── floor-coverings-wall-coverings.csv
│   └── metal-and-steel-construction.csv
│
└── results/                  # Stage 2 output (auto-generated)
    ├── metal-and-steel-construction-1.csv
    ├── metal-and-steel-construction-2.csv
    └── ...
```

---

## ✨ Features

- **Two-stage pipeline**: First collects all listing URLs, then scrapes detailed profiles — cleanly separated concerns.
- **Stealth mode**: Uses `playwright-stealth` to avoid bot detection on targeted site.
- **Cookie-based session**: Loads saved browser cookies to maintain an authenticated/verified session.
- **City-level pagination**: Iterates through every city listed under a category, paginating until all links are collected.
- **Rich data extraction**: Captures 20+ fields per business including phone types (Telephone, Mobile, WhatsApp, Fax), social links, working hours, languages spoken, services offered, payment terms, logo, and images.
- **Parallel scraping**: Both stages use `ThreadPoolExecutor` or `threading.Thread` for concurrent execution.
- **Encoding fixer**: Dedicated `encoding.py` utility to repair `latin-1` → `utf-8` mojibake in output CSVs.

## 🚀 Getting Started

### Step 1 — Extract & Save Cookies

Run `cookies.py` to open a real Chrome window, browse local.ch to establish a session, then save cookies:

```bash
python cookies.py
```

> A Chrome browser will open. Navigate to [local.ch](https://www.local.ch) if needed, then press **Enter** in the terminal to save cookies and exit.

This creates `cookies.pkl`, which all other scripts load automatically.

---

### Step 2 — Collect Listing URLs

Run `links.py` to scrape all business listing URLs for each category, organized by city:

```bash
python links.py
```

This saves CSVs to the `./links/` folder, one file per category:

```
./links/plasterer.csv
./links/floor-coverings-wall-coverings.csv
./links/metal-and-steel-construction.csv
```

Each CSV has two columns: `title` (city name) and `Link` (listing URL).

---

### Step 3 — Scrape Business Profiles

Run `local.py` to read the collected URLs and extract full business data:

```bash
python local.py
```

Input CSVs are split across threads (e.g., `1.csv`, `2.csv`, ...) under a keyword subfolder. Results are saved to `./results/<keyword>-<thread_num>.csv`.

---

### Step 4 (Optional) — Fix Encoding Issues

If output CSVs contain garbled characters (mojibake), run `encoding.py`:

```bash
python encoding.py
```

Update `path_to_read` and `path_to_save` variables inside the script before running.

## ⚙️ Configuration

### Adding New Categories (links.py)

```python
th4 = threading.Thread(target=main, args=("electricians", "https://www.local.ch/en/categories/e/electricians"))
th4.start()
```

### Adding More Input Files (local.py)

Uncomment or add entries in the `files` list:

```python
files = [
    f"./{keyword}/1.csv",
    f"./{keyword}/2.csv",
    f"./{keyword}/6.csv",   # uncomment to include
    f"./{keyword}/7.csv",
]
```

Match `max_workers` to the number of files for full parallelism.

## 🏗️ Architecture Overview

```
cookies.py
    └── Captures session cookies via undetected-chromedriver → cookies.pkl

playwright_assistant.py
    └── PlaywrightAssistantClass          # Reusable base browser controller
        ├── __init__()                    # Load cookies, launch stealth Chromium
        ├── navigate()                    # Go to URL (networkidle)
        ├── navigate_with_zoomout()       # Go to URL + zoom to 25%
        ├── extract_single_element()      # Get one element's text or attribute
        ├── extract_multiple_elements()   # Get list of element texts/attributes
        └── click_on_btn()               # Click a button by selector

links.py
    └── LocalCHLinks(PlaywrightAssistantClass)
        ├── extract_city_links()          # Get all city links for a category
        └── extract_page_links()          # Get all listing links on current page
    └── main(keyword, url)               # Iterate cities → paginate → save links
    └── Runs 3 threads in parallel

local.py
    └── HandWorker(PlaywrightAssistantClass)
        ├── clean_text()                  # Normalize whitespace and special chars
        └── extract_data_points()         # Extract all 22 fields from one listing
    └── main(th_num, keyword, input_csv) # Read links CSV → scrape → save results
    └── Runs up to 5 threads via ThreadPoolExecutor

encoding.py
    └── fix_mojibake()                   # Re-encode latin-1 strings as utf-8
    └── Reads CSV → fixes → saves clean CSV
```

## ⚠️ Notes & Limitations

- **Stealth mode** (`playwright-stealth`) is applied to every page to reduce bot detection risk.
- **Headless mode is disabled** — browser windows will be visible during scraping.
- Input CSVs for Stage 2 (`local.py`) must be pre-split into numbered files (`1.csv`, `2.csv`, ...) inside a folder named after the keyword.
- Scraping local.ch may violate their [Terms of Service](https://www.local.ch/en/terms-of-use). Use responsibly.
- `encoding.py` must have `path_to_read` and `path_to_save` set manually before running.
